Installing and configuring Nginx

Requirments

The following pre-requisites are needed for you to install and configure Nginx

  • Ubuntu: a version higher than 16
  • Command line usage
  • Nginx: latest version better
  • A working server, can either be docker or a local machine
  • Not necessary but a working domain

    • Run the following command from the command line:

    sudo apt update
    sudo apt install nginx

    Configuration

    After installation Nginx has things called server blocks, essentially they are what you will use to provide static content to your website

    After installation a directory will be created at /var/www/html.
    This is where your server blocks will be included, including their individual configuration files

    Create a simple file called index.html at the /var/www/html, this will be your test file Include the following text

    This is a test file

    The next step is to create a symbolic link of your server block to the nginx enabled sites

    Navigate to /etc/nginx/sites-enabled and from here create a symbolic link to your configuration file that exists in sites-available folder

    Tip: The sites enabled folder is used to enable server blocks while sites available is used to store both inactive and active sites

    Testing

    Now your site should be active, just incase open your config file that should exist in the sites available folder and esnure it follows the following standard:

    events { # configuration of connection processing }
    http { # Configuration specific to HTTP and affecting all virtual servers
    server {
    # configuration of HTTP virtual server 1
    location /html { # configuration for processing URIs starting with '/html' listen 80; server_name __; }
    }
    server { # configuration of HTTP virtual server 2 } }
    stream { # Configuration specific to TCP/UDP and affecting all virtual servers server { # configuration of TCP virtual server 1 } }

    Finally run sudo service nginx restart to restart the server

    FAQ

    Common Problems

    I'm getting a 404 error

    Ensure that the port the nginx is listening to is free

    Server refuses to restart

    Run sudo nginx -t to test the config file, more often than not its a syntax error

    For more go to the offical nginx documentation
